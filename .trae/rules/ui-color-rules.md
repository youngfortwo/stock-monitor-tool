# UI 配色规则

## 核心原则

页面背景必须为浅色/白色系，所有可见元素（输入框、下拉框、按钮、标签、badge）**不得使用深色/黑色背景**，确保在任何浏览器中文字可读、视觉清晰。

## 背景色规则

### 禁止使用的背景色
- `#000000`、`#111`、`#222`、`#333`、`#444`、`#555` — 任何深灰黑色系
- `#1f1f2e`、`#2a2a3a`、`#3a2d2d`、`#2d3a2d` 等暗色主题色
- 任何 HSL lightness < 40% 的纯色背景（除非文字是白色且对比度 ≥ 4.5:1）

### 必须使用浅色背景的元素
| 元素 | 背景色 | 文字色 | 边框色 |
|------|--------|--------|--------|
| 输入框 / 搜索框 | `#fff` | `#172033`（深蓝灰） | `#cbd5e1` |
| 下拉框 `<select>` | `#fff` | `#172033` | `#cbd5e1` |
| 普通按钮 | `#f1f5f9`（浅灰） | `#334155` | `#cbd5e1` |
| 主操作按钮 | `#0969da`（品牌蓝） | `#fff` | 无 |

### 标签 / Badge 颜色规范

| 含义 | 背景 | 文字 | 示例 |
|------|------|------|------|
| 看涨 / 利好 / 通过 | `#dcfce7`（浅绿） | `#166534` | 双底(W底) |
| 看跌 / 利空 / 风险 | `#fee2e2`（浅红） | `#991b1b` | 圆弧顶 |
| 中性 / 整理 / 提示 | `#fef3c7`（浅琥珀） | `#92400e` | 箱体震荡 |
| 信息 / 行业 / 概念 | `#e0e7ff`（浅靛） | `#3730a3` | 半导体 |
| 过渡态标签 | gradient light | 深色文字 | 不得用纯黑底 |

## 页面整体色调
- 页面 body 背景：`#f6f8fb`（浅灰蓝）
- 卡片/表格背景：`#fff`（纯白）
- Header 背景：`#12213a`（深蓝，仅此一处可用深色）

## 其他 UI 规则
- CSS class 统一管理颜色，避免内联 style 硬编码色值
- 新加入的组件应复用已有 class（`.concept-tag`、`.pass`、`.fail`、`.muted` 等）
- 不用 emoji 作为唯一状态指示，配合文字或颜色双通道表达
